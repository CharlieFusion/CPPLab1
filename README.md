# Лабораторная работа №1: Сравнение алгоритмов сортировки

## Техническое задание

### 1. Введение
Настоящее техническое задание определяет требования к программе на языке C++ для сравнения эффективности различных алгоритмов сортировки. Программа предназначена для демонстрации работы алгоритмов, автоматического и ручного тестирования, а также измерения времени выполнения на последовательностях различной длины и структуры.

### 2. Цель и задачи
**Цель:** Разработать программный продукт, позволяющий наглядно сравнить производительность и корректность реализации двух алгоритмов сортировки на абстрактном типе данных «последовательность».

**Задачи:**
- Реализовать интерфейс `ISorter` и два алгоритма сортировки, суммарная сложность которых в баллах не менее 14.
- Создать абстрактный тип данных `Sequence<T>` с двумя вариантами хранения: на основе массива и на основе связного списка.
- Обеспечить возможность выбора алгоритма, ручного или автоматического формирования входных данных, просмотра исходной и отсортированной последовательности.
- Реализовать функцию измерения времени выполнения сортировки и сравнения алгоритмов.
- Покрыть реализованные алгоритмы набором автоматических тестов для проверки корректности.
- Предоставить консольный интерфейс для взаимодействия с пользователем.

### 3. Требования к программе

#### 3.1 Функциональные требования
1. **Алгоритмы сортировки** – программа должна содержать не менее двух различных алгоритмов сортировки. В текущей реализации выбраны:
   - Быстрая сортировка (Quick Sort) – 10 баллов.
   - Шейкерная сортировка (Cocktail Sort / Shaker Sort) – 5 баллов.
   Сумма баллов = 15, что соответствует минимальному требованию (≥14).
2. **Тип данных «последовательность»**:
   - Абстрактный класс `Sequence<T>` с методами доступа к элементам (`Get`, `Set`), получения длины (`GetLength`), добавления элемента (`Append`, `Prepend`, `InsertAt`), удаления элемента (`Remove`), получения первого, последнего элемента и подпоследовательности (`GetFirst`, `GetLast`, `GetSubsequence`).
   - Две конкретные реализации:
     - `ArraySequence<T>` – на основе динамического массива.
     - `ListSequence<T>` – на основе циклического двусвязного списка.
3. **Интерфейс сортировки**:
   - Базовый класс `ISorter<T>` с чисто виртуальными методами `Sort(Sequence<T>* seq)` и `GetName()`.
   - Каждый алгоритм реализуется в отдельном классе-наследнике, переопределяющем `Sort` и возвращающем имя алгоритма.
4. **Режимы ввода данных**:
   - Ручной ввод последовательности с клавиатуры.
   - Автоматическая генерация последовательности заданной длины:
     - случайные числа в заданном диапазоне;
     - отсортированная последовательность;
     - обратно отсортированная последовательность;
     - все элементы равны.
5. **Режимы проверки**:
   - Вывод исходной и отсортированной последовательности на экран.
   - Автоматическая проверка корректности (проверка упорядоченности и сохранения мультимножества элементов).
6. **Измерение времени**:
   - Измерение времени выполнения каждого алгоритма с использованием высокоточного таймера (`std::chrono`).
   - Возможность сравнения времени работы двух алгоритмов на одних и тех же входных данных.
7. **Тестирование**:
   - Автоматические тесты для каждого алгоритма, включающие:
     - пустую последовательность;
     - последовательность из одного элемента;
     - два элемента;
     - уже отсортированную последовательность;
     - обратно отсортированную последовательность;
     - все элементы равны;
     - случайную последовательность;
     - почти отсортированную последовательность;
     - последовательности с отрицательными числами;
     - последовательности с дубликатами;
     - работу с `ArraySequence` и `ListSequence`;
     - большие последовательности (до 10 000 элементов) для оценки производительности.
   - Проверка результата с помощью функций `IsSorted` и `HasSameElements`.

#### 3.2 Нефункциональные требования
- **Язык реализации:** C++17 или новее.
- **Стиль кода:** соблюдение единого стиля именования (CamelCase для классов, snake_case для переменных), отступы 4 пробела, декомпозиция сложных функций.
- **Обработка ошибок:** корректная работа с пустыми последовательностями, проверка границ индексов.
- **Производительность:** алгоритмы должны быть реализованы эффективно (например, быстрая сортировка работает за O(n log n) в среднем, шейкерная – O(n²), что соответствует их природе).
- **Расширяемость:** возможность добавления новых алгоритмов путём наследования от `ISorter`.

### 5. Описание реализованных алгоритмов

#### 5.1 Быстрая сортировка (Quick Sort)
- **Принцип:** рекурсивное разделение последовательности относительно опорного элемента (pivot). Элементы меньшие pivot помещаются слева, большие – справа.
- **Сложность:** O(n log n) в среднем, O(n²) в худшем случае.
- **Особенности реализации:** опорным элементом выбран последний элемент, используется функция `Partition` для разбиения.

#### 5.2 Шейкерная сортировка (Cocktail Sort)
- **Принцип:** модификация пузырьковой сортировки, при которой проходы выполняются поочерёдно слева направо и справа налево. Это позволяет «всплывать» лёгким элементам быстрее.
- **Сложность:** O(n²) в худшем и среднем случае.
- **Особенности реализации:** оптимизирована за счёт запоминания последней позиции обмена, что позволяет сокращать границы проходов.

### 6. Структуры данных
- **`Sequence<T>`** – шаблонный абстрактный класс, задающий контракт для последовательностей:
  - `Get(int index) const` – доступ по индексу.
  - `Set(int index, TElement item)` – изменение элемента.
  - `GetLength() const` – получение размера.
  - `Append(TElement iteme)` – добавление в конец.
  - `Prepend(TElement item)` – добавление в начало.
  - `InsertAt(int index, TElement item)` – добавление в конец.
- **`ArraySequence<T>`** – хранит элементы в динамическом массиве; обеспечивает доступ за O(1), вставка в конец амортизированно O(1).
- **`ListSequence<T>`** – хранит элементы в узлах двусвязного списка; доступ по индексу O(n), вставка в конец O(1).

### 7. Пользовательский интерфейс
Консольный интерфейс предоставляет следующие возможности (реализуется в `main.cpp`):
- Выбор алгоритма сортировки (1 – Quick Sort, 2 – Cocktail Sort).
- Выбор типа последовательности (массив или список).
- Выбор способа заполнения:
  1. Ручной ввод элементов.
  2. Автоматическая генерация:
     - случайные числа;
     - почти отсортированные;
     - обратно отсортированные;
     - все одинаковые.
- Ввод длины последовательности (для автоматического режима) и диапазона значений.
- Вывод исходной и отсортированной последовательности на экран.
- Замер времени выполнения.
- Сравнение двух алгоритмов на одинаковых данных с выводом времени.

### 8. Тестирование
Тесты реализованы в файле `sorter_tests.cpp`. Они охватывают:
- Граничные случаи (пустая последовательность, один элемент).
- Различные типы входных данных (случайные, отсортированные, обратные, с дубликатами, отрицательные числа).
- Проверку на разных реализациях последовательности (`ArraySequence` и `ListSequence`).
- Тестирование на больших объёмах (до 10 000 элементов) с измерением времени.
- Сравнительное тестирование производительности для набора размеров (100, 500, 1000, 2000, 5000).

Все тесты используют `assert` для автоматической проверки корректности.
